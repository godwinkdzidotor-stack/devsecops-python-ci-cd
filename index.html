DevSecOps Python CI/CD Pipeline

A security-focused CI/CD pipeline that automatically validates Python applications with syntax checks, Bandit SAST, and Docker image builds on every push to GitHub.

GitHub Actions Bandit SAST Docker Build Security Gate in CI
Repository
github.com/godwinkdzidotor-stack/devsecops-python-ci-cd
Python GitHub Actions Secure SDLC
Stage 1 ‚Äì Syntax & Basic Tests
Quality Gate
  • Runs python -m py_compile to catch syntax errors.
  • Easy to plug in pytest or unittest for deeper checks.
  • Prevents broken code from reaching later security stages.
Python Static Checks
Stage 2 ‚Äì Bandit SAST Scan
Security Gate
  • Runs Bandit recursively against the repository.
  • Flags insecure APIs, weak crypto, hard-coded secrets.
  • Publishes an HTML report as an artifact.
Bandit SAST
Stage 3 ‚Äì Docker Build
Containerization
  • Builds a Docker image from the application code.
  • Validates that the app can run in a container.
  • Sets the stage for registry pushes and deployment.
Docker Immutable Artifacts
Security Posture
DevSecOps
  • Security checks run on every push and pull request.
  • Build can be configured to fail on high-risk Bandit findings.
  • Pipeline logs and artifacts become evidence of security testing.
Extensibility
Future Growth
  • Add dependency scanning (e.g., pip-audit or safety).
  • Integrate container scanning (Trivy, etc.).
  • Attach deploy stages to staging or production environments.
How to Reuse This Workflow
Getting Started
  1. Fork or clone the repository on GitHub.
  2. Review the workflow under .github/workflows/devsecops.yml.
  3. Adjust project name, Python version, and Bandit settings as needed.
  4. Push code or open a PR to trigger the pipeline.
  5. Inspect job logs and the Bandit HTML report under the Actions tab.
